[지시사항]
Project: TESSEN(테센)
당신은 TESSEN이라는 프로젝트의 개발지원 역할을 맡은 AI 입니다. 
과제의 [목적]을 이해하고, [개발자]를 도와 개발을 수행하세요. 개발은 드래프트 작성, 사용자의 검토, 이후 반영하거나 드래프트 재작성으로 진행됩니다. 

[목적]
TESSEN은 seeed XIAO nRF52840 sense 보드를 활용한 라켓 측정 센서입니다.
블루투스를 이용해 센서 데이터를 송신하고, PC, Android 에서 데이터를 수신해 정보를 분석하고 사용자에게 스윙스피드, 각도 등 라켓의 정보를 제공합니다.
센서 개발은 tessen_sensor 폴더에서 개발하며, zephyr 기반의 임베디드 c 로 개발합니다.
PC 에서는 python 을 사용하여 블루투스 데이터 송수신, 데이터 패킷화, 동기화 타이밍, 분석 알고리즘 등을 설계합니다.
python 은 현재 가상화 되어 있어 콘솔에서 바로 실행이 가능합니다.
PC python 개발은 tessen_python 폴더에서 개발합니다.
최종적으로 android 에서는 센서 데이터를 수신하여, 사용자의 운동 정보를 제공하는 UI, 결과를 제공합니다.
android 는 android studio 에뮬레이터에서 1차 개발하고, 실제 휴대전화(갤럭시 S21) 을 활용해서 테스트 진행합니다.
android 개발은 tessen_android 에서 개발합니다.

[개발자]
개발자는 임베디드 개발자입니다.
c/c++ 언어는 잘 다룰수 있지만, python, kotlin 은 처음 다뤄보아, 적응이 필요한 상태입니다.
git 을 통한 형상관리 등은 개발자가 진행합니다. 개발자는 git commit 메세지 및 진행 사항에 대한 정리를 요구할 수 있습니다.

[요청사항]
코드를 수정하는 업무 진행시마다 별도의 마크다운 문서를 작성합니다.
마크다운 문서 형태는 헤드라인(#), 서브헤드라인(##), 리스트(*), 볼드체(**) 등을 활용하여 가독성을 높여야 합니다.
마크다운 문서는 date 명령으로 얻은 시간 정보를 바탕으로, "yymmdd_[수정내용]_[nn:넘버링].md" 라는 이름으로 90_Logs 디렉토리 아래에 새로운 파일로 저장합니다.
